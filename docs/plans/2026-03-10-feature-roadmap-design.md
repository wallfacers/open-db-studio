## 背景与目标

open-db-studio 定位为 AI-Native Database Client，复刻 chat2db 核心功能并在
AI 精度上深度创新。核心差异点：引入 GraphRAG 知识图谱 + 业务指标层，实现
高精度 text-to-SQL，消除 LLM 幻觉。

**四阶段目标**：
- MVP：能用（连接 + 查询 + 基础 AI）
- V1：好用（完整 DB 管理 + 数据流转 + AI 增强）
- V2：专业（GraphRAG + 指标层 + 跨数据源迁移）
- V3：生态（向量库 + 插件系统 + 团队协作）

## V2 阶段（Q4 2026）—— 专业版

### GraphRAG 知识图谱引擎

| 功能 | 说明 |
|------|------|
| Schema 实体图构建 | 表、列、外键、索引解析为图节点和边，存入内置 SQLite |
| 关系路径推断 | 自动发现 JOIN 路径，注入 SQL 生成上下文 |
| 图谱可视化 | ERD 叠加业务语义标注，节点可搜索 |

### 业务指标层

| 功能 | 说明 |
|------|------|
| 指标定义 | 名称、所属表/字段、聚合函数、过滤条件、业务含义描述 |
| AI 生成草稿 | AI 扫描 Schema + 数据样本，自动推断指标草稿 |
| 用户审核确认 | 草稿编辑/批准/拒绝，确认后入库为知识节点 |
| 指标检索增强 | 提问时检索相关指标，指标定义注入 prompt |

### 高精度 Text-to-SQL 管道

```
用户问题
  → 指标检索（匹配业务含义）
  → GraphRAG 路径查询（找 JOIN 路径）
  → Schema 上下文构建（相关表+列+约束）
  → Prompt 组装 → LLM 生成 SQL
  → SQL 语法校验 → 返回编辑器
```

### 跨数据源迁移

**SeaTunnel 评估结论**：SeaTunnel Engine 依赖 JVM，Tauri 桌面端打包 JRE 增重约
150MB，偏重。V2 采用 Rust 原生实现基础迁移；V3 提供 SeaTunnel 作为可选外部引擎。

| 功能 | 说明 |
|------|------|
| 结构迁移 | DDL 跨方言转换（类型映射、约束转换） |
| 数据迁移 | 分批读写，进度展示，错误行跳过 + 报告 |
| 迁移预检 | 迁移前检查兼容性问题 |
| 迁移任务管理 | 任务列表，支持暂停/恢复/重试 |

### 完整 RAG 管道（GraphRAG + 向量双路融合）

```
用户问题
  → 向量检索（历史 SQL few-shot）

  → GraphRAG 路径查询（关联路径）
  → Schema 上下文
  → 融合 Prompt → LLM → SQL 校验 → 编辑器
```

## 关键架构约定

- GraphRAG 图谱数据存入内置 SQLite（新增 graph_nodes / graph_edges 表）
- 指标定义存入内置 SQLite（新增 metrics 表）
- 向量索引 V3 阶段由 Milvus Lite 管理，通过 Rust FFI 或 HTTP 接入
- 跨数据源迁移任务状态存入内置 SQLite（新增 migration_tasks 表）
- SeaTunnel V3 仅作为外部引擎，不嵌入 Tauri 应用包
